# eventribe — Event Management

**eventribe** (_a platform that connects people with shared interests_) empowers users and admins to discover, manage, and participate in events. It features a public-facing interface for discovery and registration, along with a dedicated admin panel for comprehensive management.

---

## Features

### User & Public Features

- **Event Discovery:** Browse the complete list of events, search, and apply filters.
- **Event Management (CRUD):** Full Create, Read, Update, and Delete (CRUD) capabilities for user-created events.
- **Registration System:** Securely register for or unregister from `events/page.tsx`, `unregister-event/route.ts`] and view all your registrations on the `My Events` page.
- **Full Authentication:** Robust authentication process including sign-up, login, and password reset.
- **Account Management:** Update profile details, delete the account, and manage payment methods.
- **Payment Integration:** A modal and APIs are in place for event payments.
- **Modern UX:** Responsive design, light/dark mode, cookie consent banner, and toast notifications for user feedback.

### Admin Panel

- **Dashboard:** A central dashboard to visualize participation statistics.
- **Event Management (CRUD):** Full Create, Read, Update, and Delete (CRUD) capabilities for all events.
- **User Management:** View all users and manage their roles (e.g., promote to admin).
- **Registration Management:** Oversee and manually manage event registrations for all users.

---

## Tech Stack

- **Framework:** Next.js (with App Router)
- **Language:** TypeScript
- **UI:** React, Tailwind CSS
- **Authentication:** NextAuth.js
- **Database:** Supabase (PostgreSQL)
- **UX :** Figma
- **UI & UX Components:** Heroicons, AOS (Animate On Scroll)
- **Backend Services:** Next.js API Routes, Email Sending Service (with `lib/email-templates/`)

---

## Project Architecture

The project is structured for maximum maintainability and clarity using modern Next.js conventions.

- **`src/app`**: Uses the Next.js App Router. Routing, layouts, and pages are defined here.
- **Route Groups**: Uses groups like `(main)` and `(auth)` to structure public and authentication layouts without affecting URL segments.
- **Admin Section**: `src/app/admin` is a dedicated route group containing the entire admin panel.
- **API Backend**: `src/app/api` hosts all backend logic (API Routes), neatly organized by function (auth, admin, account, etc.).
- **Business Logic**: `src/lib` isolates core logic: `data-access` (DB queries), `services` (like email), and auth configuration.
- **UI Library**: `src/ui` contains all reusable React components (buttons, modals, cards), organized by feature.

## Getting Started

Follow these steps to get the project running locally.

### Prerequisites

Make sure you have Node.js installed on your machine.

### Steps

1.  **Clone the repository:**

    ```bash
    git clone https://github.com/Only-tech/eventribe-nextjs.git
    cd eventribe-nextjs
    ```

2.  **Install dependencies:**

    ```bash
    npm install
    # or
    yarn install
    ```

3.  **Run the development server:**

    ```bash
    npm run dev
    # or
    yarn dev
    ```

4.  **Open your browser:**
    Navigate to [http://localhost:3000](http://localhost:3000) to see the application.

> **Note:** This project is connected to a Supabase database that can be restored from `dataTable.sql`. Make sure to set up your local environment variables in a `.env.local` file.

# eventribe — Gestion d'Événements

**eventribe** (_rassemble des personnes partageant des centres d'intérêts communs_) permet aux utilisateurs et aux administrateurs de découvrir, gérer et participer à des événements. Il comprend une interface publique pour la découverte et l'inscription, ainsi qu'un panel d'administration dédié pour une gestion complète.

---

## Fonctionnalités

### Fonctionnalités Utilisateurs & Publiques

- **Découverte d'Événements :** Parcourez la liste complète des événements, recherchez et appliquez des filtres.
- **Gestion des Événements (CRUD) :** Capacités complètes de Créer, Lire, Mettre à jour et Supprimer (CRUD) pour les événements créés par l'utilisateur.
- **Système d'Inscription :** Inscrivez-vous ou désinscrivez-vous en toute sécurité/page.tsx`, `unregister-event/route.ts`] et consultez toutes vos inscriptions sur la page "Mes Événements".
- **Authentification Complète :** Processus d'authentification robuste incluant l'inscription, la connexion et la réinitialisation du mot de passe.
- **Gestion de Compte :** Mettez à jour les détails du profil, supprimez le compte et gérez les moyens de paiement.
- **Intégration de Paiement :** Une modale et des API sont en place pour le paiement des événements (la logique de prélèvement réel n'est pas mise en place).
- **UX Moderne :** Design responsive, mode clair/sombre, bannière de consentement aux cookies et notifications (toasts) pour le feedback utilisateur.

### Panel Administrateur

- **Tableau de Bord :** Un dashboard central pour visualiser les statistiques de participation.
- **Gestion des Événements (CRUD) :** Capacités complètes de Créer, Lire, Mettre à jour et Supprimer (CRUD) pour tous les événements.
- **Gestion des Utilisateurs :** Visualisez tous les utilisateurs et gérez leurs rôles (ex: promotion en admin).
- **Gestion des Inscriptions :** Supervisez et gérez manuellement les inscriptions aux événements pour tous les utilisateurs.

---

## Technologies Utilisées

- **Framework :** Next.js (avec App Router)
- **Langage :** TypeScript
- **UI :** React, Tailwind CSS
- **Authentification :** NextAuth.js
- **Base de Données :** Supabase (PostgreSQL)
- **UX :** Figma
- **Composants UI & UX :** Heroicons, AOS (Animate On Scroll)
- **Services Backend :** Next.js API Routes, Service d'envoi d'email (avec templates `lib/email-templates/`)

---

## Architecture du Projet

Le projet est structuré pour une maintenabilité et une clarté maximales en utilisant les conventions modernes de Next.js.

- **`src/app`** : Utilise l'App Router de Next.js. Le routage, les layouts et les pages y sont définis.
- **Groupes de Routes** : Utilise les groupes comme `(main)` et `(auth)` pour structurer les layouts publics et d'authentification sans affecter les segments d'URL.
- **Section Admin** : `src/app/admin` est un groupe de routes dédié qui contient l'ensemble du panel d'administration.
- **API Backend** : `src/app/api` héberge toute la logique backend (API Routes), soigneusement organisée par fonction (auth, admin, account, etc.).
- **Logique Métier** : `src/lib` isole la logique principale : `data-access` (requêtes BDD), `services` (comme l'envoi d'emails), et la configuration de l'authentification.
- **Bibliothèque UI** : `src/ui` contient tous les composants React réutilisables (boutons, modales, cartes), organisés par fonctionnalité.

## Installation

Suivez ces étapes pour démarrer le projet localement.

### Prérequis

Assurez-vous d'avoir Node.js installé sur votre machine.

### Étapes

1.  **Clonez le dépôt :**

    ```bash
    git clone https://github.com/Only-tech/eventribe-nextjs.git
    cd eventribe-nextjs
    ```

2.  **Installez les dépendances :**

    ```bash
    npm install
    # ou
    yarn install
    ```

3.  **Démarrez le serveur de développement :**

    ```bash
    npm run dev
    # ou
    yarn dev
    ```

4.  **Ouvrez votre navigateur :**
    Rendez-vous sur [http://localhost:3000](http://localhost:3000) pour voir l'application.

> **Note :** Le projet fonctionne avec une base de données liée à supabase qui peut être restaurée à partir de `dataTable.sql`. Assurez-vous de configurer vos variables d'environnement locales dans un fichier `.env.local`.
